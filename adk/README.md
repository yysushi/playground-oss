- google/sdk-go vs a2aproject/a2a-go
a2a-go provides the protocol-level plumbing, while adk-go provides the agent framework on top of it.

a2a-go is the Go SDK for the A2A (Agent2Agent) protocol. Its purpose is to help developers build A2A-compliant agent servers and clients, with support for transports like gRPC, REST, and JSON-RPC.  ￼

adk-go, by contrast, is a higher-level Go framework for building, evaluating, and deploying AI agents. Google describes it as an open-source, code-first toolkit for sophisticated AI agents.
