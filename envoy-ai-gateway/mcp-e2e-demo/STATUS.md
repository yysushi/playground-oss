## ⚠️ Current Issue

The AI Gateway MCP proxy architecture requires a **sidecar container** in the gateway pods that acts as an MCP protocol translator. This is not being automatically injected.

**Error:** `503 Service Unavailable` when calling `/mcp` endpoint through gateway

**Root Cause:** The AI Gateway controller creates a Backend pointing to `192.0.2.42:9856` (MCP proxy sidecar) but this sidecar isn't running.

## 📋 Next Steps to Complete

### Option 1: Wait for AI Gateway MCP Feature Maturity
The MCP Gateway feature is very new (documented as of June 2025 spec). The sidecar injection mechanism may require:
- Specific EnvoyGateway configuration
- MCP proxy container image specification
- Pod annotations for sidecar injection

### Option 2: Simpler HTTP Proxy Approach
Instead of using MCPRoute, use a standard HTTPRoute to proxy to the MCP server:

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: mcp-http-route
spec:
  parentRefs:
  - name: mcp-gateway
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /mcp
    backendRefs:
    - name: mcp
      port: 80
```

This would work immediately but loses:
- OAuth integration via MCPRoute
- MCP-specific features (tool filtering, server multiplexing)

### Option 3: Manual Sidecar Configuration
Research and manually configure the MCP proxy sidecar in gateway pods.
