## 🔧 What Works (Verified)

```bash
# OAuth metadata endpoints
curl -s http://localhost:8080/.well-known/oauth-protected-resource/mcp | jq
curl -s http://localhost:8080/.well-known/oauth-authorization-server/mcp | jq

# MCP session initialization + tool listing (example JWT)
TOKEN=$(curl -s https://raw.githubusercontent.com/envoyproxy/ai-gateway/main/examples/mcp/mcp_oauth_example.yaml | \
  awk '/# test token:/{print $4}')
SESSION=$(curl -s -D - http://localhost:8080/mcp -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "mcp-protocol-version: 2025-06-18" \
  -d '{"jsonrpc":"2.0","method":"initialize","id":1}' | \
  awk '/^mcp-session-id:/{print $2}' | tr -d '\r')
curl -s -X POST http://localhost:8080/mcp \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "mcp-protocol-version: 2025-06-18" \
  -H "Mcp-Session-Id: $SESSION" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":2}'
```

## 📋 Notes

- OAuth validation uses a local JWKS ConfigMap seeded during setup.
- The test script uses the official example JWT to validate MCPRoute OAuth.
- To use a real OAuth server for tokens, ensure the token issuer matches `securityPolicy.oauth.issuer`.
